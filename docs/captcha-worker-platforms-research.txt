HAR FILE ANALYSIS — 2CAPTCHA TRAINING FLOW
===============================================================================

From the 3 existing HAR files:

TRAINING API ENDPOINTS:
- GET  /api/v1/training/v2/current-step?hostname=2captcha.com
- POST /api/v1/training/v2/current-step (submit answer or skip)
- GET  /api/v1/training/v2/summary (check completion status)

TRAINING IMAGE URLS (STATIC — ANSWERS ARE CONSTANT):
- /assets/worker-training/image67.png
- /assets/worker-training/image5.png
- /assets/worker-training/image82.png
- /assets/worker-training/skip.jpeg
- /assets/worker-training/level-10/captcha-05.jpg  (organized by level!)

KEY FINDING: Training images use FIXED filenames organized by level
(e.g., level-10/captcha-05.jpg). This means:
- Same filename = same captcha image = same correct answer
- Answers are CONSTANT, not random
- We can build a LOOKUP TABLE: image_filename → correct_answer
- Once we capture correct answers from the HAR, future training
  is instant replay with zero OCR needed

RESPONSE CODES:
- 204 = Correct answer (step completed)
- 422 = Wrong answer (Param "answer" is invalid)
- 200 = GET returns step data including:
  - isLevelCompleted, isTrainingCompleted
  - mustSkip (true = auto-skip this step)
  - hint (e.g., "Here are 4 digits that need to be written")
  - errorHint (shown after wrong answer)
  - data.captcha_id, data.params, data.textinstructions

WORKER API (captcha_api.php):
- action=login&thash=KEY  → Login, get balance/reputation
- action=getbot&thash=KEY → Poll for captchas (returns captcha data or errors)
- action=sendrecaptchabot → Submit answer (with salt = md5(prefix + captcha_id))
- action=closerecaptchabot → Skip captcha (avoid timeout/ban)

RUCAPTCHA USES SAME API:
- https://rucaptcha.com/captcha_api.php works with same thash
- Same account, same balance, same everything
- Just different domain (Russian-language frontend)
